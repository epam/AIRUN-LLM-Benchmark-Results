        *   `header`: (string) Page header content.
        *   `subheader`: (string) Page subheader content.
        *   `body`: (string) Main body content of the page.
        *   `url`: (string) URL of the page.
        *   `type`: (string) Page type (e.g., template name).
        *   `published`: (string) Publication status ('Y', 'N', 'schedule').
        *   `published_date`: (string/number) Date the page was published (or scheduled to be published).
        *   `themePages`: (array) List of available page templates for the current theme.
        *   `timestamp`: (string/number) Timestamp of the last modification.
        *   `extras`: (object)  Holds additional, custom data associated with the page.
        *   `misc`: (object) A catch-all for any other miscellaneous data.

#### 2.3. `Users` Factory (`users.js`)

*   **Feature:** Global User Data Storage
    *   **Implementation:** Similar to the `Page` factory, it provides a shared object to store user information.
    *   **Dependencies:** None.
    *   **Properties:**
        *   `id`: (string/number) User's unique identifier.
        *   `username`: (string) User's login name.
        *   `name`: (string) User's full name.
        *   `bio`: (string) User's biographical information.
        *   `email`: (string) User's email address.
        *   `facebook`: (string) User's Facebook profile URL (optional).
        *   `twitter`: (string) User's Twitter profile URL (optional).
        *   `photo`: (string) URL of the user's profile photo.
        *   `role`: (string) User's role (e.g., 'admin', 'editor').

#### 2.4. `pageCtrl` Controller (`pageCtrl.js`)

*   **Feature:** Page Creation and Editing
    *   **Implementation:**  Handles user interactions within the `page.html` template.  It uses the `REST`, `Page`, and `Users` factories to manage data and interact with the backend.
    *   **Dependencies:** `$scope`, `REST`, `$location`, `Page`, `$rootScope`, `$routeParams`, `$upload`, `Users`, `$translate`.
    *   **Key Functions:**
        *   `localVersion()`:  Restores a previously saved, local version of the page from `localStorage`.
        *   `deleteNewerVersion()`:  Clears the locally saved version of the page from `localStorage`.
        *   `deletePage()`: Deletes the current page and its associated data (revisions, extras, tags) via the `REST` factory.
        *   `updatePageType()`: Updates the page type in the `Page` factory.
        *   `titleChange()`:  Handles changes to the page title, auto-generating the URL if necessary.
        *   `descriptionChange()`: Handles changes to the page description.
        *   `urlChange()`: Handles changes to the page URL.
        *   `saveLocal()`: Saves the current page data to `localStorage`.
        *   `autocompleteTags()`: Provides tag suggestions based on user input.
        *   `selectSuggestion()`:  Adds a selected tag suggestion to the page's tags.
        *   `savePage(duplicate)`:  Saves or updates the page.  Handles both creating new pages and updating existing ones.  Also manages page revisions and extras.  The `duplicate` parameter determines whether to create a new page with the same content.
    *   **Local Storage:** Uses `localStorage` to provide a basic "draft" feature, allowing users to recover unsaved changes.
    *   **Error Handling:** Includes basic error handling for saving and updating pages, displaying messages to the user via `$rootScope.$broadcast('notify', ...)`.
    *   **Tag Management:**  Implements tag autocomplete and saving/deletion of tags.
    *   **Revision Management:** Creates revisions of the page whenever it's saved.
    *   **Extras Management:**  Handles saving and deleting "extras" (custom data fields) associated with the page.

#### 2.5 page.html

*   **Newer Version Notification:** Displays a notification if a newer version of the page is found in local storage.  Provides buttons to discard, compare, or use the local version.
*   **Top Bar:** Contains navigation links and a title.
*   **Action Bar:** Includes buttons for deleting, duplicating, and saving the page.  Also handles the confirmation prompt for deleting a page.
*   **Page Editor Form:**
    *   **Type Selection:** A dropdown to select the page type (template).
    *   **Title Input:**  A text input for the page title, with a character counter.
    *   **Description Input:** A textarea for the page description, with a character counter.
    *   **Tags Input:** A text input for tags, with autocomplete suggestions.
    *   **URL Input:** A text input for the page URL.
    *   **Publish/Draft/Schedule Radio Buttons:**  Allows the user to choose the publication status of the page.  Includes a date/time picker for scheduling.
*   **Accessibility Features:**
    *   Uses `label` elements with `for` attributes to associate labels with form controls.
    *   Uses `translate` attribute for internationalization.
    *   Uses `ng-show` and `ng-hide` to manage the visibility of elements, which can be helpful for screen readers.
