### 1. Component Overview

This documentation details the Angular.js components responsible for managing pages within the "cosmo" application.  These components provide functionality for creating, reading, updating, and deleting (CRUD) pages, along with features for version control, scheduling, and content editing. The core components are `REST` factory, `Page` factory, `Users` factory, and the `pageCtrl` controller, working in conjunction with the `page.html` template.  This system interacts with a RESTful backend API to persist page data.  The components are designed to be integrated into a larger content management system (CMS).

### 2. Component Features

**2.1 REST Factory (rest.js)**

*   **Purpose:**  Provides a centralized interface for interacting with the backend REST API.  It leverages Angular's `$resource` service to simplify HTTP requests.
*   **Implementation:** The `REST` factory returns an object containing `$resource` instances for various API endpoints (blocks, comments, content, files, menus, modules, sitemaps, themes, settings, users). Each `$resource` instance is pre-configured with the base URL and parameters.  The `update` method for each resource is explicitly set to use the `PUT` HTTP method.
*   **Dependencies:**  Angular's `$resource` service and the `Page` factory.

**2.2 Page Factory (page.js)**

*   **Purpose:**  Acts as a global store for page-related data.  It holds default values and allows for data sharing between different parts of the application.
*   **Implementation:** The `Page` factory returns an object with properties representing common page attributes (id, title, description, url, type, publish, scheduleDate, themePages, timestamp, extras, misc).  This object is used to initialize the `pageCtrl` and to store temporary data during editing.
*   **Dependencies:** None.

**2.3 Users Factory (users.js)**

*   **Purpose:** Stores data about the currently logged-in user.
*   **Implementation:** The `Users` factory returns an object with properties representing user attributes (id, username, name, bio, email, facebook, twitter, photo, role).
*   **Dependencies:** None.

**2.4 pageCtrl Controller (pageCtrl.js)**

*   **Purpose:**  Handles the logic for the page management interface.  It manages user interactions, data binding, API calls, and page saving/updating.
*   **Implementation:**
    *   **Initialization:**  Initializes the `$scope.page` object with data from the `Page` factory.  Sets up initial values for schedule date and theme pages.
    *   **Version Control:** Implements a basic version control mechanism using `localStorage` to detect and handle unsaved changes.  Provides functionality to revert to the previously saved version or discard the newer version.
    *   **CRUD Operations:**  Uses the `REST` factory to perform CRUD operations on page data.  Includes functions for saving, updating, deleting, and duplicating pages.
    *   **Content Editing:**  Handles user input for page title, description, URL, and publish status.  Includes auto-generation of URLs from titles.
    *   **Tag Management:**  Provides autocomplete functionality for tags, using the `REST` factory to query the backend for suggestions.
    *   **Scheduling:**  Allows users to schedule pages for publication at a future date and time.
    *   **Extras Management:** Handles additional page data stored in the `extras` property.
*   **Dependencies:** `$scope`, `REST`, `$location`, `Page`, `$rootScope`, `$routeParams`, `$upload`, `Users`, `$translate`.
