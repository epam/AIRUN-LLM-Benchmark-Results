## Application Documentation: Cosmo Page Editor

This document provides a comprehensive overview of the Cosmo Page Editor application, built using Angular.js. It details the functionality, user interactions, business requirements, technical considerations, and data management strategies.

**1. Application Overview**

The Cosmo Page Editor is a web application designed for creating, editing, and managing content pages within a content management system (CMS). It allows users to define page titles, descriptions, URLs, content type, publication status, and associated tags.  The application interacts with a RESTful backend API to persist and retrieve page data.

**2. Component Architecture and Data Flow**

*   **Modules:** The application utilizes Angular modules (`cosmo`) to organize its components.
*   **Factories:**
    *   `REST`:  Handles communication with the backend API using the `$resource` service. Provides access to various endpoints for managing blocks, requirements, comments, content, files, menus, modules, sitemaps, themes, settings, and users.
    *   `Page`:  Stores page-related data (id, title, description, url, type, publish status, schedule date, tags, theme pages) in a central location.  Acts as a shared data model.
    *   `Users`: Stores current user data (id, username, name, bio, email, etc.).
*   **Controller:** `pageCtrl` is the main controller responsible for handling user interactions, managing the page data, and interacting with the `REST` and `Page` factories.
*   **Data Flow:**
    1.  User interacts with the UI (e.g., edits a field).
    2.  `pageCtrl` updates the `Page` factory with the new data.
    3.  `pageCtrl` uses the `REST` factory to communicate with the backend API to save or update the page data.
    4.  The backend API processes the request and updates the database.
    5.  The backend API returns a response.
    6.  `pageCtrl` handles the response and updates the UI accordingly.
*   **Templates:** `page.html` defines the user interface, utilizing Angular directives (e.g., `ng-model`, `ng-click`, `ng-show`) to bind data and handle events.

**3. User Interaction Patterns**

*   **UI Components:**
    *   **Form Fields:** Text inputs, text areas, select dropdowns, radio buttons, and date/time pickers for editing page properties.
    *   **Buttons:**  Save, Duplicate, Delete, Publish/Draft/Schedule, Compare, Use, Discard.
    *   **Select Dropdown:**  For choosing the page type, populated from `page.themePages`.
    *   **Tag Input:**  Allows users to enter and manage tags associated with the page.  Includes autocomplete functionality.
    *   **Confirmation Dialog:**  Appears before deleting a page.
    *   **Newer Version Alert:**  Displays a warning if a newer version of the page exists in local storage.
*   **Workflows:**
    *   **Creating a New Page:**  Navigate to `/new`, fill in the form fields, and click "Save".
    *   **Editing an Existing Page:** Navigate to the page's URL, modify the form fields, and click "Save".
    *   **Deleting a Page:** Click "Delete", confirm the deletion in the dialog, and click "Yes".
    *   **Publishing/Drafting/Scheduling:** Select the desired publication status using radio buttons and optionally set a schedule date.
    *   **Tag Management:** Enter tags, use autocomplete suggestions, and select from the list.
*   **Form Validation:**
