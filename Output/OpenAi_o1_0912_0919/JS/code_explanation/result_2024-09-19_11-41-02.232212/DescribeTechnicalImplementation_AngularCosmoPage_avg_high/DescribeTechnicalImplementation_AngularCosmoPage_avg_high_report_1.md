# Page Component Documentation

## Overview

The **Page Component** is a crucial part of the application, designed to facilitate the creation, editing, and management of content pages within the frontend application. It provides a user interface that allows users to input and modify page details such as the title, description, URL, tags, and publication settings. The component integrates with AngularJS controllers and services to handle data binding, form submissions, and communication with the backend RESTful API.

## Features

- **Create and Edit Pages**: Users can create new pages or edit existing ones.
- **Form Elements**: Includes form fields for title, description, URL, tags, and publication status.
- **Type Selection**: Allows selection of page templates based on available theme pages.
- **Tag Autocompletion**: Provides tag suggestions to enhance user experience.
- **Publication Scheduling**: Offers options to publish immediately, save as draft, or schedule for future publication.
- **Local Storage for Unsaved Changes**: Detects unsaved local versions and allows users to restore or discard them.
- **Confirmation Prompts**: Includes confirmation dialogs for critical actions like deleting a page.
- **Responsive Design**: Adaptable to different screen sizes and devices.
- **Accessibility Considerations**: Utilizes proper labels and input associations to support assistive technologies.

## Props

The component accepts the following properties (scope variables) through its AngularJS controller:

| Prop Name            | Type            | Description                                                                     | Required |
|----------------------|-----------------|---------------------------------------------------------------------------------|----------|
| `page.id`            | Number          | Unique identifier of the page.                                                  | No       |
| `page.title`         | String          | Title of the page.                                                              | Yes      |
| `page.description`   | String          | Description or summary of the page content.                                     | Yes      |
| `page.url`           | String          | URL slug for the page.                                                          | Yes      |
| `page.type`          | String          | Template or type of the page (from available theme pages).                      | Yes      |
| `page.publish`       | String (`'Y'`, `'N'`, `'schedule'`) | Publication status of the page.              | Yes      |

| `page.tags`          | Array of Strings| List of tags associated with the page.                                          | No       |

> **Note**: All props are managed within the controller `$scope` and the `Page` factory, which stores page-related data globally.
